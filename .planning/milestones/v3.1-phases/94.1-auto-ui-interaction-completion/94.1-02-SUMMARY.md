# Phase 94.1 Plan 02: ActionPanel Reactivity + D-02 Footer Rule Summary

**One-liner:** Switched ActionPanel to reactive controller computeds (currentChoices.value, validElements.value) and replaced the per-choice notation walk in allCurrentChoicesAnchored with `validElements.value.length > 0` (D-02), updating 4 stale assertions and adding 2 vacuous-case tests.

## What Was Built

Fixed two foundational interaction defects that blocked Go Fish and Checkers playability:

1. **Bug C fix (ActionPanel.vue):** `filteredChoices` and `filteredValidElements` now read the REACTIVE controller computeds (`actionController.currentChoices.value` and `actionController.validElements.value`) rather than the non-reactive helper functions (`getCurrentChoices()`, `getValidElements(pick)`). This aligns ActionPanel with the board bridge which already used `currentChoices.value`.

2. **D-02 footer rule (useActionController.ts):** Replaced the `allCurrentChoicesAnchored` per-choice notation walk with a single-statement body: `return validElements.value.length > 0`. The footer is now suppressed ONLY when the board offers clickable element picks (validElements non-empty), not for notation choices. Choice picks always keep the footer.

3. **Interaction test harness (ActionPanel.interaction.test.ts):** Two mounted-component tests using the real useActionController and real ActionPanel under jsdom, exercising the full controller→component render chain.

### Deviation 1 (Research Assumption Wrong) — Test C1 NOT RED before Bug C fix

**Found during:** Task 1
**Issue:** The plan expected Test C1 (Bug C reactivity) to fail (RED) against current code because the research claimed "Vue 3 does not track plain Map access." Empirical verification showed Vue 3.5's reactive Proxy DOES intercept `Map.get()` calls and track them as reactive dependencies inside `computed()` getters. When `actionSnapshot.value.pickSnapshots.set(...)` is called, the Proxy triggers effects for any computed that previously read `.get()` on that Map.

This means `filteredChoices` (which calls `getCurrentChoices()` → `actionSnapshot.value.pickSnapshots.get(...)`) IS reactive to Map mutations in Vue 3.5. Bug C does not manifest in tests — the choices appear correctly after async fetch even before the fix.

**Impact:** Test C1 was GREEN before and after the fix (not RED→GREEN as planned). The fix (using `currentChoices.value`) is still valuable as it makes the reactive dependency chain explicit and follows the pit-of-success principle.

**Test C1 still provides protection** against regression: if a future refactor uses `shallowRef` for actionSnapshot or removes the Proxy tracking, the test will catch it.

**Deviation type:** Research-assumption deviation (not a code deviation)

### Deviation 2 (Additive) — Two vacuous-case tests added for element/elements picks

**Found during:** Task 3
**Action:** The plan specified updating the element/elements pick tests to "keep `toBe(true)` by adding fetchPickChoices, AND add a sibling test for the vacuous case." Two sibling tests were added (one for element, one for elements) asserting `toBe(false)` when no fetchPickChoices is provided. These are additional tests beyond the plan's strict "4 stale assertions updated" count.

**Rationale:** The plan explicitly called for sibling vacuous-case tests ("Additionally add a sibling test asserting `toBe(false)` for an element pick with NO valid elements"). Net result: 4 stale assertions updated + 2 new sibling tests added.
