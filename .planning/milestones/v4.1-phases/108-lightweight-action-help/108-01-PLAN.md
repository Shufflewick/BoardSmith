<objective>
Thread a single new display-only string field — `help?: string` — through the four type/propagation layers below the UI: engine `ActionDefinition`, the action DSL builder, session `ActionMetadata` + both `buildActionMetadata`/`buildSingleActionMetadata` paths, and the UI `ActionMetadata` composable type. This is the authoring substrate for HELP-01: an author attaches help text once, and it arrives at both UI paths for free via the existing action-metadata broadcast.

Purpose: Establish the parity-safe propagation chain (one source of truth: `ActionDefinition.help`) before any UI consumes it. Locking the contract in Wave 1 lets Waves 2-3 build against a tested type surface.
Output: `help` field + `.help()` builder + propagation through session + UI type, with engine unit coverage and a session propagation integration test covering BOTH build paths.
</objective>
