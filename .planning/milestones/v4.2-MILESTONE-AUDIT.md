## Post-ship fixes (2026-06-30, after v4.2 tag)

User browser-testing the go-fish tutorial surfaced two real tutorial-substrate/parity bugs the v4.2 verification missed (root cause: `simulateTutorial` drives action args directly, bypassing the UI auto-fill + board-click path, and the browser checks happened to use the action panel on a fresh seat-1 turn):

1. **Flow not reset on tutorial start** (BoardSmith `tutorial-controller.ts`) — launching the tutorial mid-game (opponent's turn) dealt the preset but left the turn on the opponent, so the learner had no action (empty panel, dead clicks). Fix: `TutorialController.start()` calls `game.startFlow()` after `setup()` to restart at the learner's turn. Also fixes the same latent bug for checkers. Regression test added (`tutorial-controller.test.ts`). BoardSmith 1709 green.
2. **Board card-click dead — opponent not auto-filled** (go-fish `tutorial.ts`) — the ask-for-rank step used action-wide `suppressAutoFill`, blocking the single-opponent `target` auto-fill, so the action stayed on 'target' and clicking the highlighted 7 on the board (which only fires on the 'rank' pick) did nothing. Fix: `suppressAutoFillFor: 'rank'` scopes suppression to the rank. UI board-path parity test added (`GoFishAsk.interaction.test.ts`). go-fish 78 green.

Both fixed + tested; live browser re-confirmation of the board click pending (extension was disconnected at fix time). Lesson: tutorials need a UI-level board-click/auto-fill test, not just `simulateTutorial`.
