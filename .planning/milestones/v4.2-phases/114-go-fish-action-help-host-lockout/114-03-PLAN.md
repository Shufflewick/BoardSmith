go-fish App.vue uses GameShell's DEFAULT #action-panel — it does NOT set `suppress-action-panel` and does NOT override the #action-panel slot. So GameShell renders the substrate `ActionPanel` (BoardSmith src/ui/components/auto-ui/ActionPanel.vue) in the bottom action dock, which contains `ActionHelpPopover` and shows the "?" driven by `actionMetadata['ask'].help` + `isActionHelpVisible`. The custom GameTable.vue only renders the board region (cards/hands). => the ask "?" is EXPECTED to surface via the substrate dock panel once 114-01 lands; a GameTable.vue change is likely NOT needed.

Contrast with Phase 112-04: that gap was the custom board omitting `anchorAttrs` for OVERLAY anchoring — a different surface. Action help renders in the dock ActionPanel, which go-fish already uses. Verify, don't assume a gap.

Dev host lockout flag — `boardsmith dev --lock-teaching`:
  sets DevHostConfig.teachingDisabled → broadcasts state.teachingDisabled → ControlsMenu gate
  `v-if="(showHint !== undefined || hasTutorial) && !teachingDisabled"` hides hint/demo/heatmap/tutorial.
  "Show action help" lives in the Play group, gated by hasActionHelp (NOT teachingDisabled) — stays visible.
