## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| game iframe → outer dev-host page | GameShell (untrusted-of-outer) postMessages game state to `window.parent`; DevHost consumes it. |
| outer page window → agent/tooling | `window.__BOARDSMITH_DEVTOOLS` exposes read-only game state to any script on the dev-host page. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-119-04 | Information Disclosure | `window.__BOARDSMITH_DEVTOOLS` global leaking game state into production | mitigate | Both sender (GameShell watcher) and receiver/global (DevHost onMounted) gated by `import.meta.env.DEV`; Vite dead-code-eliminates the branch in production builds so the global is never assigned. Made an explicit acceptance criterion ("absent from production build"). |
| T-119-05 | Tampering | malicious postMessage injecting a fake devtools snapshot | mitigate | DevHost `onWindowMessage` filters `data.source === 'shufflewick-game'` before handling; the new case sits inside that guard. Dev-host is a local-only tool (localhost:5173), not internet-exposed. |
| T-119-06 | Information Disclosure | perspective leak (hidden info) via `getState` | accept | `state.value?.state` is the Phase 117 perspective-aware broadcast already filtered server-side for the seat; no additional hidden info is exposed by re-broadcasting it to the same-seat dev view. |

No other trust boundary is introduced by this plan.
