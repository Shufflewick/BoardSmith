GameHistory.vue: watcher lines 40-70 (the bug — startIndex = processedMessages.value.length,
re-adds from index 0 after clear). clearHistory() lines 91-94 (sets processedMessages=[]
AND messageCounter=0 — the reset is what enables re-add). copyHistory() lines 98-125.
Copy/Clear buttons: template lines 170-183 (the `header-buttons` block, sidebar mode only —
sheet mode has no such buttons). props.messages: Array<string | {text,type?}>.

THE FIX (research §Pitfall 1 + Code Examples): introduce `let lastProcessedSourceIndex = 0`.
The watcher iterates `for (i = lastProcessedSourceIndex; i < newMessages.length; i++)` and
sets `lastProcessedSourceIndex = newMessages.length` at the end. clearHistory() empties
processedMessages but MUST NOT reset lastProcessedSourceIndex (and need not reset
messageCounter). This way cleared messages are never re-added; only genuinely new ones append.
