<objective>
Define the Phase 104 reserved `content` slot on `TutorialStep` and `TutorialStepView`,
narrowing both from `unknown` to the annotation content model: an array of annotations,
each a `{ text; target?; placement? }`, where `AnnotationTarget` is a discriminated union
of board-element | action | panel. Board targets reuse the existing `ElementRef`
(`{id,name,notation}`) so the parity contract (`matchesRef` precedence) is the only selector.

Purpose: This is the foundational data contract every other Phase 105 plan binds to. The
projection helper `getActiveTutorialStepView` (gate.ts) already copies `step.content` through
verbatim, so narrowing the type makes the annotation shape flow to the client with zero new
runtime code — the change is type-level plus a guard test.
Output: Narrowed `content` types + a test proving the shape survives the projection.
</objective>
