<objective>
Build the single `TutorialOverlay.vue` layer and extend `BoardMessage.vue` so a tutorial step's
`content` renders as a text bubble plus a targeted highlight ring — locating targets ONLY through
the shared `data-bs-*` anchors from Plan 02. Per 105-CONTEXT Area 2/3 + 105-UI-SPEC §1–§5:

- The overlay injects `gameState`, reads `state.value.state.tutorial.content`, and renders only
  when content is present (`v-if` → zero markup/cost in normal play).
- For each annotation: resolve its target — element (id>notation>name via data-bs-el-*),
  action (data-bs-action), or panel (data-bs-panel) — measure its rect, draw a ring (3px accent
  stroke + glow + marker chip + connector caret) over it, and render the bubble (BoardMessage,
  extended with an annotation variant: left-aligned prose, --bsg-r-md card, caret, anchor-rect placement).
- Graceful: target querySelector null → bubble-only fallback (top/bottom), no ring, no throw,
  re-measure on next tick. Honor placement rules + reduced-motion + z-20 layering from the UI-SPEC.

Purpose: This is the visible deliverable for criteria #1 and #2 (bubble + highlight; targets
cell/piece/panel/action). It is ONE component over the shared anchor — never per-renderer styling —
which is what makes the Plan 05 parity test possible.
Output: TutorialOverlay.vue + BoardMessage annotation variant + a component test.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| author content → overlay DOM | Authored annotation `text` is rendered into the bubble |
| AnnotationTarget → querySelector | Author-supplied ref/actionName builds a DOM selector |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-105-05 | Tampering (XSS) | bubble text rendering | mitigate | Render `text` via Vue text interpolation / slot text only — never `v-html`/innerHTML. (Acceptance grep below.) |
| T-105-06 | Denial of service | querySelector with author values | mitigate | Build attribute selectors via `CSS.escape()` on id/notation/name/actionName so crafted values cannot break the selector or match outside scope; scope all queries to `.boardregion`. |
| T-105-07 | Tampering | ring blocking input | mitigate | ring layer is `pointer-events:none` — decorative, never intercepts the underlying control. |
