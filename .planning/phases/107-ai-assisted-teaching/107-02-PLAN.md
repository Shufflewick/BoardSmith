Build the session-layer transient teaching substrate and the player-triggered hint (AI-01) and heatmap (AI-03) query APIs. Teaching state is injected into broadcast state AFTER `buildPlayerState()` runs (keeping that pure function untouched) and is never persisted.

Purpose: `buildPlayerState()` is a pure function with no session reference; hint/heatmap/narration are session-only transient aids that must not enter engine serialization or break checkpoint/replay determinism.
Output: New `hint`/`heatmap`/`narration` fields + `HeatmapEntry` type on `PlayerGameState`; `requestHint`/`clearHint`/`setHeatmapVisible` public methods; broadcast injection; auto-clear on action and on state replacement.
