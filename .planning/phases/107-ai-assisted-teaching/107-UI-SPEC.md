## Spacing Scale

All spacing uses the existing `--bsg-s*` token series. New overlay components must reference tokens, not literals.

| Token | Value | Usage |
|-------|-------|-------|
| `--bsg-s1` | 4px | Icon gaps, chip internal padding |
| `--bsg-s2` | 8px | Ring offset, bubble gap from target, caret gap |
| `--bsg-s3` | 12px | Narration card vertical padding, chip top/left anchor |
| `--bsg-s4` | 16px | Narration card horizontal padding, bubble horizontal padding |
| `--bsg-s5` | 24px | Section spacing in ControlsMenu Teaching group |
| `--bsg-s6` | 32px | Not used in this phase |

Exceptions:
- Heatmap chip minimum size: 24px × 24px (legibility floor for badge text — not a spacing value).
- Hint ring offset: 3px outside target rect (matches existing TutorialOverlay `OFFSET = 3` — not a spacing token; keep as literal per existing pattern).

Source: `src/ui/theme.ts` `STATIC_TOKENS`.

## Color

All color references use Slate token variables. No literals in component files (project hard-rule).

| Role | Token (dark value) | Usage |
|------|--------------------|-------|
| Dominant (60%) | `--bsg-bg` (#121417) / `--bsg-surface` (#1c2026) | Board background, overlay base |
| Secondary (30%) | `--bsg-surface-2` (#232831) / `--bsg-surface-3` (#2b313b) | Narration card, hint bubble, chip background base |
| Accent (10%) | `--bsg-accent` (#1fb8a6 dark / #0d9488 light) | See reserved list below |
| Destructive | `--bsg-danger` | Not used in this phase — no destructive actions |

Accent (`--bsg-accent`) reserved for:
1. Hint ring border (`border: 3px solid var(--bsg-accent)`) — reuses TutorialOverlay ring exactly
2. Hint ring glow / spread — reuses TutorialOverlay `box-shadow` exactly
3. Marker chip background (`--bsg-accent` fill, `--bsg-accent-ink` text) — reuses TutorialOverlay `.bsg-tutorial-ring__chip` exactly
4. Heatmap chip tint: `color-mix(in srgb, var(--bsg-accent) {intensity}%, transparent)` where intensity = `15 + round(normalizedValue × 70)` (range 15%–85%)
5. Heatmap best-move chip solid border: `2px solid var(--bsg-accent)` on the single highest-value chip

All other chips use: `border: 1px solid color-mix(in srgb, var(--bsg-accent) {intensity + 15}%, transparent)`.

Text color on heatmap chips:
- Intensity ≥ 55 (normalizedValue ≥ 0.57): `var(--bsg-accent-ink)` (dark ink on teal)
- Intensity < 55: `var(--bsg-ink)` (light ink on semi-transparent)

Non-color cues (WCAG 2.2 AA 1.4.1 — no color-only signalling):
- Hint ring: the "i" marker chip (shape + glyph) identifies the ring as an AI hint.
- Heatmap: the numeric percentage badge is the primary signal; chip opacity/tint is secondary.
- Heatmap best-move: the 2px solid border provides a shape distinction independent of color.
- Demo narration: the "AI Demo" prefix label identifies the message source.

## Component Inventory

Three new UI components land in `src/ui/components/helpers/`:

### 1. HintOverlay.vue

Renders a single transient annotation (the move hint) using the same visual primitives as `TutorialOverlay`.

- Mount: `<Teleport to="body">` — `position: fixed; inset: 0; z-index: 20; pointer-events: none`
- Reads from: `gameState.state.hint?.annotation` (a single `Annotation` object, not an array)
- Rendering: identical to `TutorialOverlay` — renders one ring + one bubble using the same CSS classes (`.bsg-tutorial-ring`, `.bsg-tutorial-ring__chip`, `BoardMessage variant="annotation"`)
- Target resolution: same `buildSelector` logic as `TutorialOverlay` — resolves `data-bs-el-id` anchors via `document.querySelector`
- Z-index: 20 — same stratum as `TutorialOverlay`; both may coexist (tutorial step + hint may render simultaneously during Phase 109 authored tutorials)
- Motion: fade-in `var(--bsg-dur-base)` on appear; 2-iteration glow pulse (matches TutorialOverlay). `prefers-reduced-motion`: no animation.
- Accessibility: ring and marker chip `aria-hidden="true"`; bubble uses `BoardMessage` `role="status"` `aria-live="polite"` (announced on appear)

### 2. HeatmapOverlay.vue

Fixed-position overlay rendering per-cell evaluation chips from MCTS stats.

- Mount: `<Teleport to="body">` — `position: fixed; inset: 0; z-index: 15; pointer-events: none; overflow: visible`
- Reads from: `gameState.state.heatmap?.entries` — array of `{ cellRef: ElementRef; normalizedValue: number; isBest: boolean }`
- Shown when: `gameState.state.heatmap?.visible === true`
- Chip positioning: per entry, resolve `data-bs-el-id` anchor via `document.querySelector`, compute chip center at `rect.left + rect.width/2`, `rect.top + rect.height/2`; apply `transform: translate(-50%, -50%)`

**Chip visual spec:**

| Property | Spec |
|----------|------|
| Shape | Circle — `border-radius: var(--bsg-r-pill)` |
| Size | `min(floor(min(rect.width, rect.height) * 0.68), 52px)` — minimum 24px |
| Background | `color-mix(in srgb, var(--bsg-accent) {15 + round(normalizedValue × 70)}%, transparent)` |
| Border (non-best) | `1px solid color-mix(in srgb, var(--bsg-accent) {intensity + 15}%, transparent)` |
| Border (best) | `2px solid var(--bsg-accent)` |
| Text | `round(normalizedValue × 100)` + "%" — e.g. "73%" |
| Text font | `var(--bsg-font)`, `var(--bsg-text-xs)`, weight 600, `--bsg-line-tight` |
| Text color | intensity ≥ 55 → `var(--bsg-accent-ink)`; otherwise `var(--bsg-ink)` |
| Pointer events | `none` |
| Aria | `aria-hidden="true"` — decorative layer |

**Chip appear animation:** `opacity: 0 → 1` over `var(--bsg-dur-base)` `var(--bsg-ease)`. All chips animate together (no stagger). `prefers-reduced-motion`: instant display, no animation.

- Observers: `ResizeObserver` on `.boardregion` + each resolved target — re-measures on board resize or scroll (same pattern as `TutorialOverlay`).

### 3. DemoNarrationMessage.vue (or `BoardMessage variant="narration"`)

Displays the pre-move announcement during the AI demo. Implemented as a new `'narration'` variant on the existing `BoardMessage` component.

- Position: `position: 'top'` — `top: var(--bsg-s3)` from the board top edge, horizontally centered. Does not conflict with the existing turn-prompt pill at the bottom.
- Visual:
  - Background: `var(--bsg-surface-3)` (slightly more prominent than the standard `--bsg-surface-2` pill)
  - Border: `1px solid var(--bsg-line-2)`
  - Radius: `var(--bsg-r-md)` (card shape, distinguishes from the pill turn-prompt)
  - Padding: `var(--bsg-s2) var(--bsg-s4)` (vertical × horizontal)
  - Max-width: `min(280px, 90%)`
  - Box-shadow: `var(--bsg-shadow-sm)`
- Text layout: inline — "AI Demo" prefix in `var(--bsg-ink-2)` followed by the move string in `var(--bsg-ink)`, both `--bsg-text-sm` weight 400
- `role="status"`, `aria-live="polite"` — announces each move for screen reader users
- Z-index: 10 (above turn-prompt z-5, below heatmap z-15, below TutorialOverlay z-20)
- Animation: fade-in `var(--bsg-dur-base)` on appear; component unmounts after the announce delay fires. `prefers-reduced-motion`: instant.

## Teaching Controls Placement

Teaching controls surface inside `ControlsMenu` as a new "Teaching" group. Controls are only rendered when the game has an AI configured (`GameDefinition.ai` is defined and non-null).

**New ControlsMenu props:**

| Prop | Type | Notes |
|------|------|-------|
| `showHint` | `boolean \| undefined` | Renders "Get a hint" item; `undefined` hides the group |
| `hintDisabled` | `boolean` | True when not at a player decision point |
| `isDemoRunning` | `boolean` | Toggles "Watch AI demo" / "Stop demo" label |
| `isHeatmapVisible` | `boolean` | Drives `aria-checked` on heatmap toggle |

**New ControlsMenu emit:** `'teaching-action'` with payload `'hint' | 'demo-toggle' | 'heatmap-toggle'`.

**Menu items (rendered only when `showHint` is defined):**

```
[Teaching group label]
[ ] Get a hint             ← role="menuitem"; disabled when hintDisabled
[ ] Watch AI demo          ← role="menuitem"; label swaps to "Stop demo" when isDemoRunning
[✓] Show move quality      ← role="menuitemcheckbox"; aria-checked reflects isHeatmapVisible
```

Group header label: "Teaching" — uses the existing `.grouplabel` CSS class from `ControlsMenu`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Hint CTA (ControlsMenu item) | "Get a hint" |
| Hint CTA disabled label | "Get a hint" (disabled state — no label change) |
| Hint bubble default text | "Suggested move" |
| Demo start CTA | "Watch AI demo" |
| Demo stop CTA | "Stop demo" |
| Narration format | "[Player name]: [move description]" — e.g. "AI: move c3 → d4" |
| Heatmap toggle | "Show move quality" (label is stable; `aria-checked` indicates state) |
| Teaching group label | "Teaching" |
| Hint unavailable (no AI) | Teaching section absent — no disabled copy needed |
| Error: MCTS hint fails | Toast: "Hint unavailable — the AI could not suggest a move." |
| Error: Demo AI fails mid-game | Narration message: "Demo stopped." — then demo exits cleanly |
| Empty board state (heatmap with no candidates) | Heatmap overlay renders nothing — no empty state message needed |

Narration text derivation (default): `${playerName}: ${actionName} ${args summary}`. Game author may supply a custom formatter via the `onBeforeMove` session hook (CONTEXT.md Area 2).

## Accessibility Checklist (WCAG 2.2 AA)

| Criterion | Implementation |
|-----------|----------------|
| 1.4.1 Use of Color | Heatmap: numeric % badge is primary signal; chip tint is secondary. Hint ring: "i" chip glyph. Best-move: 2px solid border shape cue. |
| 1.4.3 Contrast (text) | Chip badge text: `--bsg-accent-ink` (dark on teal at high intensity) or `--bsg-ink` (light at low intensity) — both ≥ 4.5:1. Narration text: `--bsg-ink` on `--bsg-surface-3` — ≥ 4.5:1. Verified by existing `theme.contrast.test.ts`. |
| 2.1.1 Keyboard | Teaching controls in `ControlsMenu` inherit existing keyboard navigation (arrow keys, Escape, Enter/Space on menuitem). Overlay elements are all `pointer-events: none` — never intercepting keyboard. |
| 4.1.3 Status Messages | Narration: `role="status"` `aria-live="polite"`. Hint bubble: `role="status"` via `BoardMessage`. Heatmap chips: `aria-hidden="true"` (decorative — value conveyed numerically in text). |
| Reduced motion | All three overlays: no animation under `prefers-reduced-motion: reduce`. Instant display/hide. |
