<objective>
Build the single shared reveal surface for HELP-01: a new `ActionHelpPopover.vue` (Teleport-to-body, fixed-positioned, `role="tooltip"`, caret, reduced-motion guard, outside-click/Escape/mouse-leave dismiss) plus its `?` trigger button, and wire it into `ActionPanel.vue` by wrapping each action button in a `.action-btn-group` and rendering the popover when the global toggle is on and content exists. ActionPanel gains two props: `isActionHelpVisible` and `disabledActions` (both threaded from GameShell in Plan 03).

Purpose: One reveal path serving pointer hover, touch tap, and the global toggle — set in ONE shared component so both UI paths get identical behavior (parity hard-rule). AutoUI is wired here; GameShell/ControlsMenu wiring + persistence is Plan 03.
Output: `ActionHelpPopover.vue` + component tests (content, show/hide, dismiss, reduced-motion, disabled-reason, parity fixture) and ActionPanel integration with affordance visibility tests.
</objective>

<interfaces>
ActionHelpPopover props (the contract ActionPanel and custom UIs both use):
- `actionName: string` — used to build the popover element id `${actionName}-help-tip`.
- `helpText?: string` — from `ActionMetadata.help` (Plan 01).
- `disabledReason?: string` — from `disabledActions[name]`.
- `triggerLabel: string` — the action prompt/display name, for the aria-label "Help for {triggerLabel}".

Rendering contract (UI-SPEC "Interaction States" + "Copywriting"):
- Affordance hidden entirely when neither helpText nor disabledReason is present, OR when the parent's toggle is off (ActionPanel applies the toggle gate via v-if).
- Popover content: helpText section (if present); a divider only when BOTH helpText and disabledReason exist; disabledReason section = "Why disabled:" label + reason text. Text via `{{ }}` interpolation only — never v-html (T-108 security rule, mirrors TutorialOverlay T-105-05).

ActionPanel new props (mirror existing action-metadata prop style):
- `isActionHelpVisible?: boolean` — global toggle state (default-treated truthy by GameShell; here just a gate).
- `disabledActions?: Record<string, string>` — per-action disabled reasons from PlayerGameState.
</interfaces>
