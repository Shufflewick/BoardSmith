> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Vue components, v4.0 Slate token system) |
| Preset | not applicable |
| Component library | none (hand-rolled; no Radix, no base-ui) |

## Spacing Scale

This phase uses the existing Slate spacing tokens. No new tokens are introduced.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| --bsg-s1 | 4px | Gap between "?" glyph and circle border; caret triangle half-size |
| --bsg-s2 | 8px | Popover internal padding (vertical); gap between help section and divider |
| --bsg-s3 | 12px | Popover internal padding (horizontal) |
| --bsg-s4 | 16px | Min space above/below popover before flip threshold triggers |
| --bsg-s5 | 24px | Not used in new elements |
| --bsg-s6 | 32px | Not used in new elements |

Exceptions:
- "?" affordance button: 24x24px minimum tap target (WCAG 2.2 SC 2.5.8). This is not in the scale but matches the existing `.cancel-btn` / `.clear-selection-btn` minimum tap-target precedent.
- Popover max-width: 240px (not a spacing token — a layout constraint).
- Popover caret: 6px CSS triangle (3px per axis).

## Color

Resolved from `src/ui/theme.ts` DARK_COLOR_TOKENS (dark default) / LIGHT_COLOR_TOKENS (light OS / forced).
All references below are token references, not literal values.

| Role | Token (dark value) | Usage in this phase |
|------|---------------------|---------------------|
| Dominant (60%) | var(--bsg-bg) #121417 | Shell background; no new usage |
| Secondary (30%) | var(--bsg-surface-3) #2b313b | **ActionHelpPopover background** — one step above surface-2, distinguishes the popover from surrounding surface-2 elements |
| Accent (10%) | var(--bsg-accent) #1fb8a6 | "?" affordance in open/active state; ControlsMenu toggle ON state (matches existing heatmap toggle blueprint) |
| Destructive | var(--bsg-danger) #ef7a5f | Not used in this phase |

Accent reserved for: action-btn background (existing), "?" affordance when its popover
is open (`aria-expanded="true"` state), ControlsMenu toggle pill ON state.

Neutral ink tokens for new elements:

| Token | Usage |
|-------|-------|
| var(--bsg-ink-3) | "?" affordance in idle (closed) state; "Why disabled:" section label; popover border color in line-2 |
| var(--bsg-ink-2) | Help text body copy; disabled-reason body text |
| var(--bsg-line-2) | Popover border; divider between help-text section and disabled-reason section |

Non-color cue requirement: the "?" affordance is a circled question-mark SVG glyph —
distinguishable in grayscale and for color-blind users without relying on color alone.

## New Elements — Visual Specification

### 1. Action Button Group Wrapper (`.action-btn-group`)

Each action button is wrapped in a new element that provides the positioning context
for the "?" affordance.

```
display: inline-flex
align-items: stretch
position: relative
```

This is the anchor element for the `ActionHelpPopover`. The wrapper flattens into the
dock flow because the existing `.action-buttons { display: contents }` container
propagates the flex context.

### 2. Help Affordance Button (`.action-help-btn`)

The "?" that triggers the popover on pointer-hover and touch-tap.

**Anatomy:**

```
<button
  class="action-help-btn"
  type="button"
  aria-label="Help for {action prompt}"
  aria-expanded="{true|false}"
  aria-controls="{action-name}-help-tip"
  @click="togglePopover"
  @mouseenter="showPopover"
  @mouseleave="hidePopover"
>
  <!-- inline SVG: circled question mark, 14x14px viewBox -->
  <svg width="14" height="14" viewBox="0 0 14 14" fill="none"
       stroke="currentColor" stroke-width="1.5" aria-hidden="true">
    <circle cx="7" cy="7" r="6"/>
    <path d="M7 9V8.5c0-.5.3-.9.7-1.1C8.3 7 9 6.3 9 5.5a2 2 0 0 0-4 0"
          stroke-linecap="round" stroke-linejoin="round"/>
    <circle cx="7" cy="11" r="0.5" fill="currentColor" stroke="none"/>
  </svg>
</button>
```

**Layout:**

- Position: absolute, top 0, right 0 (corner of `.action-btn-group`)
- OR: inline sibling directly after `.action-btn` if absolute positioning conflicts with
  dock wrapping — executor chooses whichever maintains 44px action-btn touch target
- Minimum tap target: 24x24px (padding: 4px 5px achieves this around a 14px icon)
- No background, no border in idle state

**States:**

| State | color | background |
|-------|-------|------------|
| Idle (closed) | var(--bsg-ink-3) | transparent |
| Hover (pointer) | var(--bsg-ink-2) | transparent |
| Open (popover visible) | var(--bsg-accent) | transparent |
| Disabled action (closed) | var(--bsg-ink-3) | transparent |

**Visibility rule:** `display: none` when global toggle is OFF. Shown whenever the
toggle is ON AND (action has help text OR action has a disabled reason).

### 3. ActionHelpPopover Component (`ActionHelpPopover.vue`)

**File:** `src/ui/components/helpers/ActionHelpPopover.vue`

**Positioning:**
- Teleported to `<body>` (same pattern as ControlsMenu popover and TutorialOverlay)
- `position: fixed; z-index: 60`
- Anchor: computed from `trigger.getBoundingClientRect()`
- Default placement: below the trigger, left-aligned to trigger left edge
- Flip threshold: if bottom edge would fall within 8px of viewport bottom, flip to above
- Left constraint: if right edge would exceed `window.innerWidth - 8px`, shift left

**Dimensions:**
- max-width: 240px
- min-width: 120px
- width: max-content (fits short strings without extra space)

**Visual:**

```css
background: var(--bsg-surface-3);
border: 1px solid var(--bsg-line-2);
border-radius: var(--bsg-r-md);     /* 11px */
box-shadow: var(--bsg-shadow-sm);
padding: var(--bsg-s2) var(--bsg-s3);  /* 8px 12px */
```

**Caret:** 6px CSS triangle (border-trick, `aria-hidden="true"`), same fill as
`var(--bsg-surface-3)`, border-color `var(--bsg-line-2)`. Points upward when popover
is below trigger; points downward when flipped above.

**Content layout (when both help text and disabled reason exist):**

```
[ help text section ]
[ 1px divider: var(--bsg-line-2), margin: var(--bsg-s2) -var(--bsg-s3) ]
[ disabled-reason section ]
```

Help text section: plain string at var(--bsg-text-sm), var(--bsg-ink-2), line-height 1.5.

Disabled-reason section:
- Label: "Why disabled:" at var(--bsg-text-xs), var(--bsg-ink-3), displayed as a block
- Reason text: at var(--bsg-text-sm), var(--bsg-ink-2), line-height 1.5

When only one section exists, no divider is rendered.

**Accessibility:**

```html
<div
  id="{action-name}-help-tip"
  role="tooltip"
  aria-live="polite"
>
  ...content...
</div>
```

The trigger button carries `aria-describedby="{action-name}-help-tip"` when the
popover is mounted. Dismiss via: outside click/tap, Escape key, trigger mouse-leave
(pointer only), trigger tap (touch: toggle).

**Motion:**

- Pointer path (hover): popover appears with `opacity 0→1, transform translateY(-2px)→translateY(0)` over `var(--bsg-dur-fast)` (120ms) `var(--bsg-ease)` on enter. Reverse on leave.
- Touch path (tap): same animation.
- `@media (prefers-reduced-motion: reduce)`: no animation — instant show/hide (opacity snap only, no transform).

### 4. ControlsMenu "Show action help" Toggle

**Placement:** Within the existing "Play" group (always visible, not AI-gated). Inserted
after the "Auto end turn" row and before "Undo last action".

**Template (mirrors heatmap toggle exactly):**

```html
<button
  class="mi"
  type="button"
  role="menuitemcheckbox"
  :aria-checked="isActionHelpVisible"
  @click="emit('teaching-action', 'help-toggle')"
>
  <!-- inline SVG: circled question mark, 16x16 viewBox -->
  <svg viewBox="0 0 24 24" aria-hidden="true">
    <circle cx="12" cy="12" r="9"/>
    <path d="M12 16v-.5c0-.8.5-1.5 1.2-1.8C14.4 13.2 15 12 15 10.5a3 3 0 0 0-6 0"
          stroke-linecap="round" stroke-linejoin="round"/>
    <circle cx="12" cy="18.5" r="0.75" fill="currentColor" stroke="none"/>
  </svg>
  Show action help
  <span class="r">
    <span class="toggle" :class="{ on: isActionHelpVisible }"></span>
  </span>
</button>
```

**Prop:** `isActionHelpVisible?: boolean` (new prop on ControlsMenu, mirrors `isHeatmapVisible`)

**Emit:** `'teaching-action': [action: 'hint' | 'demo-toggle' | 'heatmap-toggle' | 'help-toggle']`
(extend the existing union — no new emit name needed)

**Toggle pill styling:** inherits existing `.toggle` / `.toggle.on` CSS verbatim —
no new styles needed for the pill itself.

### 5. GameShell localStorage Persistence

**Key:** `boardsmith_action_help` (follows existing `boardsmith_*` prefix pattern)

**Storage value:** `'true'` or `'false'` (string, matches browser localStorage API)

**Default:** `'true'` (help is ON when key is absent)

**Read pattern** (mirrors `getPlayerName()` in `GameShell.vue:88-96`):

```ts
function getActionHelpEnabled(): boolean {
  const KEY = 'boardsmith_action_help';
  const stored = localStorage.getItem(KEY);
  return stored === null ? true : stored === 'true';
}

function setActionHelpEnabled(value: boolean): void {
  const KEY = 'boardsmith_action_help';
  localStorage.setItem(KEY, String(value));
}
```

**Scope:** single global client boolean — not per-game-type. Survives tab close and
reload.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (toggle label) | "Show action help" |
| "?" affordance aria-label | "Help for {action prompt}" (e.g., "Help for Move Piece") |
| Disabled-reason section label | "Why disabled:" |
| Empty state | Not applicable — "?" affordance is hidden entirely when there is no content to show (help text absent AND no disabled reason). No empty state message. |
| Error state | Not applicable — help text is read-only metadata; no user-authored input or network request. |
| Destructive confirmation | Not applicable — no destructive actions in this phase. |

**Disabled action with reason — full popover copy:**

```
Why disabled:
{disabledActions[name] reason string verbatim}
```

**Action with help text only — full popover copy:**

```
{actionDef.help string verbatim}
```

**Action with both — full popover copy:**

```
{actionDef.help string verbatim}
────────────────────────
Why disabled:
{disabledActions[name] reason string verbatim}
```

(The divider is a CSS `<hr>`-equivalent element, not literal dashes.)

## Interaction States — Decision Table

| Scenario | Affordance visible? | Popover content |
|----------|---------------------|-----------------|
| Toggle OFF | No | N/A |
| Toggle ON, no help, no disabled reason | No | N/A |
| Toggle ON, help text only, action enabled | Yes | help text |
| Toggle ON, help text only, action disabled | Yes | help text + divider + "Why disabled:" + reason |
| Toggle ON, no help text, action disabled, reason exists | Yes | "Why disabled:" + reason |
| Toggle ON, no help text, action disabled, no reason | No | N/A |

## Parity Contract

Both of the following paths must render identical popover content from the same data:

| Path | How help text is accessed |
|------|--------------------------|
| AutoUI (ActionPanel.vue) | `props.actionMetadata[name].help` |
| Custom UI | `actionController.getActionMetadata(name)?.help` |

Both paths get `disabledActions[name]` from the same `PlayerGameState` broadcast.
The `ActionHelpPopover` component is shared — imported and used in both paths.
No per-renderer opt-in. This is the hard parity rule from the CONTEXT.md.
