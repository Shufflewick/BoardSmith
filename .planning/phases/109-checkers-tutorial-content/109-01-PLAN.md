<objective>
Close carry-forward LR-02: make tutorial action gating per-selection-name aware. Today `getGateReasonForValue` lumps all values into one set and is blind to which selection produced them, so checkers' "piece c3 → square d4" cannot be gated independently. This plan replaces the reserved flat `from`/`to` fields on `TutorialGateAllowList` with a `selections` map keyed by selection name, threads `selection.name` into the gate, and matches each selection's value only against its own matcher.

Purpose: Unblocks CHK-01 (two-step move teaching) — the checkers tutorial (Plan 03) gates the `piece` selection by element id and the `destination` selection by `toNotation`.
Output: Extended `TutorialGateAllowList`, `SelectionMatcher` type, per-selection `getGateReasonForValue`, 3 updated call sites, unit + action-pipeline tests.
</objective>
