<objective>
Wire the `hint` and `heatmap-toggle` teaching ops through the dev-host path using the Phase 109 `start-tutorial` three-file pattern. The hint op runs the existing ephemeral bot's `play()` and builds an `Annotation` with the move's target; the heatmap op runs `playWithStats()` (only when visible=true) and builds deduped `HeatmapEntry[]`. `SnapshotSessionHost.handleOp` stores the result in the Plan 01 `transientTeachingState` and re-broadcasts via `broadcastCurrent()`; hint clears on the seat's next action and everything clears on undo.

Purpose: Two of the three AI teaching beats (hint, heatmap) become live in the dev host. Reuses the production `GameSession` semantics ported to the stateless host — not an invention.
Output: hint + heatmap ops fully wired (bridge → executeOp → host) + tests.
</objective>
