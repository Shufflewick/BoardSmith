<objective>
PIT-03: At `startFlow()`, statically walk the flow-node tree, collect every action name referenced by `action-step` / `simultaneous-action-step` nodes, and throw with an actionable message on any referenced-but-unregistered action. The reverse case (registered but referenced by no actionStep) is a `devWarn`, not a throw. Function-valued `actions` fields cannot be statically enumerated and are an explicitly documented blind spot.

Purpose: Catch typo'd / unregistered action names before the game starts, exhaustively across all flow branches, rather than only when a branch is reached at runtime.
Output: New `walkFlowNodes` generator, a `#validateActionReachability()` pass on `Game`, and PIT-03 unit tests (first `game.test.ts` in the tree).
</objective>
