<objective>
PIT-04 (rule 2 of 2): Add the `boardsmith/no-element-array-state` ESLint rule — SYNTACTIC-ONLY (matches the existing rules' plain-AST style), NOT auto-fixable. It flags `GameElement[]`-typed class fields in non-`GameElement` classes and assignment of `game.all()`/`game.first()` results to persistent class properties, steering authors to element-tree accessors at point-of-use or storing IDs.

Purpose: Element arrays stored as plain state go stale when elements move. This makes the footgun a lint error. Reuses the RuleTester harness convention established in plan 120-04.
Output: The rule, its RuleTester test, and its registration in index.ts.
