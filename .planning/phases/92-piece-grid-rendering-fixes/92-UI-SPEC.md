# Phase 92 — UI Design Contract

> Visual and interaction contract for piece & grid rendering fixes.
> Generated by gsd-ui-researcher. Source: CONTEXT.md D-01..D-05 + AutoElement.vue inspection.

---

## Design System

| Property | Value |
|----------|-------|

| Preset | not applicable |
| Component library | none (Vue 3 + scoped CSS) |

No shadcn. No Tailwind. The project uses scoped `<style>` blocks inside Vue SFCs with

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Token label (centered on disc) | 12px | 700 (bold) | 1.0 (single line, vertically centered) |
| Error panel heading | 14px | 600 (semibold) | 1.3 |
| Error panel body / prop hint | 12px | 400 (regular) | 1.5 |

Source: existing `.hex-label { font-size: 12px }` and `.piece { font-size: 0.8rem }` (AutoElement.vue).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant surface (60%) | rgba(255,255,255,0.03) | Board and space container backgrounds (existing) |
| Secondary surface (30%) | rgba(255,255,255,0.08) | Cards, sidebar, hand containers (existing) |
| Accent — selection (10%) | #00d9ff | Active selection outline on selectable elements (existing) |
| Accent — action selectable | rgba(46,204,113,0.6) | Outline on elements that can be actioned (existing) |
| Destructive / error | #e74c3c with 15% opacity fill | Error panel background; error icon tint |
| Token neutral (ownerless) | #888888 | Fill for pieces with no owner — matches hex-piece-circle neutral fill |

Accent reserved for:
- `#00d9ff` — selected element outline (`.is-selected`) only
- `rgba(46,204,113,...)` — action-selectable element outline (`.action-selectable`) only
- These must NOT be used for decorative purposes or general borders

### Token color contract (D-05)

Token fill color is determined by the following priority order:

1. **Owner present** — use `player.color` directly from the element's owning player (engine-first-class
   since v2.5). Do NOT introduce a CSS class `player-${seat}` for piece tokens; read the actual
   `player.color` string and apply it as an inline `background` / SVG `fill`.
2. **No owner** — fill with `#888888` (neutral grey, matching existing hex-piece-circle neutral).

Inner label text is always `#ffffff` (white) for legibility over any player color.
Drop shadow on token: `0 2px 8px rgba(0,0,0,0.3)` (matches existing `.piece` box-shadow).

## Component Inventory

### 1. Piece with image (`$images` present)

**Visual:** `<img>` or sprite-sheet `<div>` at the piece's natural dimensions. Fill the piece
container (width: 100%, height: 100%, object-fit: cover). Same rendering path as existing
card-image (`.card-image` pattern in AutoElement.vue ~1411).

**Template shape:**
```html
<!-- kind === 'image', src is URL -->
<img class="piece-image" :src="src" alt="" aria-hidden="true" />

<!-- kind === 'sprite' -->
<div class="piece-sprite" :style="spriteStyle" />
```

**CSS contract:**
```css
.piece-image {
  width: 100%;
  height: 100%;
  object-fit: cover;
  border-radius: 50%;        /* disc pieces keep the circle clip */
  display: block;
}
.piece-sprite {
  width: 100%;
  height: 100%;
  background-repeat: no-repeat;
  border-radius: 50%;
}
```

### 2. Piece without image (default token, D-05)

**Visual:** Colored disc (40×40px, border-radius: 50%) with centered 12px bold white label.

**Template shape:**

```html
<div class="piece-token" :style="{ background: tokenColor }">
```

**CSS contract:**

### 3. Grid error panel (D-03)

Renders **inside the board area** in place of the grid when coordinates cannot be resolved.
Does NOT throw. The rest of the UI remains usable.

**Dimensions:** Full-width of the board container, height auto (wraps content). Minimum 1 line
of heading + 1 line of hint text.

**CSS contract:**
```css

  border: 1px solid rgba(231, 76, 60, 0.5);
  border-radius: 8px;
  padding: 12px 16px;

.grid-error-panel__heading {
  font-size: 14px;
  font-weight: 600;

.grid-error-panel__hint {
  font-size: 12px;
```

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | None — phase has no user-initiated actions |
| Empty state heading | Not applicable (piece with no image renders as token, not empty) |
| Empty state body | Not applicable |
| Grid error heading | `Grid "{elementId}" can't render` |
| Grid error hint | `Declare $rowCoord and $colCoord on this Grid element` |
| Destructive confirmation | None — no destructive actions in this phase |

### Error message rules (D-03)

Error messages MUST:
- Name the element id exactly as declared in the game (e.g., `"board"`, `"mainGrid"`)
- Name the exact prop(s) to add (e.g., `$rowCoord`, `$colCoord`)
- Be written in plain English — no camelCase prop names in prose; show the prop in code style
- Use `console.error(...)` in addition to the panel (both paths required)

Error messages MUST NOT:
- Include file paths, line numbers, or stack trace fragments
- Include internal variable names or component names (`AutoElement`, `boardSize`, etc.)
- Use the word "error" in the heading — lead with what can't render and why

**Exact error string format (D-03):**
```
Grid "{id}" can't render — declare $rowCoord/$colCoord on the Grid element
```

Both the panel heading and the `console.error` call use this same string. The panel heading
may split it at the em-dash for visual clarity (heading / hint), but the full string goes
to console.

## Helper API Contract (D-01)

These are the shared composables/functions the executor must implement. Names are at the
planner's discretion per CONTEXT.md but the shapes below are the design contract.

### `resolvePieceVisual(element)`

```ts
type PieceVisual =
  | { kind: 'image'; src: string }
  | { kind: 'sprite'; sprite: string; x: number; y: number; width: number; height: number }
  | { kind: 'token'; color: string; label: string };

function resolvePieceVisual(element: GameElement): PieceVisual
```

- Returns `kind: 'image'` when `$images.face` is a bare URL string.
- Returns `kind: 'sprite'` when `$images.face` is a sprite-sheet object.
- Returns `kind: 'token'` when no image is present; `color` = `player.color` or `#888888`.
- Lifts the existing `$images` extraction logic from AutoElement.vue (~320–405) — no duplication.

### `useGameGrid(element)` (or equivalent function)

```ts
type GridResult =
  | { ok: true; rows: number; cols: number }
  | { ok: false; error: string };

function useGameGrid(element: GameElement): GridResult
```

Resolution order (D-02):
1. Explicit `$rowCoord`/`$colCoord` attributes — scan children for max values.
2. Inferred `gridCoordNames` from children — scan children for max values.
3. Neither available → `{ ok: false, error: "Grid \"{id}\" can't render — declare $rowCoord/$colCoord on the Grid element" }`.

The hardcoded `{ rows: 8, columns: 8 }` fallback is removed from every call site.

## Interaction Contract

This phase adds no new user interactions. The existing interaction substrate
(`useBoardInteraction`, drag/drop, FLIP animations) is unchanged. The piece-token and
piece-image components inherit click/drag handling from their parent `.piece` wrapper — the
wrapper div keeps its existing event bindings; only the inner content changes.

Hover state on token (carry-over from existing `.piece`):
- `transform: scale(1.05)` on `:hover` when piece is not dragging
