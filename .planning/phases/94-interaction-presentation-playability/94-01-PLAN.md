<objective>
Migrate the entire ref-highlight protocol from the singular `sourceRef`/`targetRef`/`ref`
fields to a single generalized `refs: { ref: ElementRef; role: 'source' | 'target' | 'highlight' }[]`
array (D-01 / INTERACT-03). This is an ATOMIC migration: every type layer and every consumer
changes together so the build stays green and no dual path survives (No Backward Compatibility).
The singular fields are DELETED, not preserved.

Purpose: A single anchor model that can express multiple highlighted cells per choice — the
precondition for Checkers multi-jump (all source/target/intermediate squares highlighted at once)
and for every downstream interaction/overlay plan in this phase.
Output: 7-layer type migration + 3 runtime consumers updated + 2 test files (1 new) green.
</objective>

|----------|-------------|
| game-author callback → session | `boardRefs()`/`boardRef()` author code returns refs that become wire payload |
| session → client (WebSocket) | refs array crosses to the untrusted client and drives highlighting |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-94-01-01 | Tampering | refs entries reference element IDs not in validElements | mitigate | `triggerElementSelect` validates against validElements before dispatch (existing guard, useBoardInteraction.ts:287-290); refs are highlight metadata only, not an authorization token |
| T-94-01-02 | Information Disclosure | refs array could carry element identity for elements the viewer cannot see | mitigate | refs are built in pick-handler from validElements which are already the player's legal/visible choices; no hidden-element ref is emitted because hidden elements never become valid choices |
| T-94-01-03 | Denial of Service | malformed boardRefs() callback throws during pick build | accept | try/catch suppression preserved from original; a throwing callback yields no refs rather than crashing the pick |
| T-94-01-SC | Tampering | npm/pip/cargo installs | n/a | Phase installs zero packages — no package-manager tasks in this plan |
</threat_model>
