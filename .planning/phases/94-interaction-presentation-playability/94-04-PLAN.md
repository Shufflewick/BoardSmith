<objective>
Ship the per-UI presentation overlay (D-04 / PRESENT-01/02/03): a plain object map
(class/name/attribute → { image, label, stats, render }) declared in a sibling file, passed as a
`:presentation` prop, provided down the GameShell → AutoUI → AutoRenderer → renderers chain, and
resolved AFTER engine visibility filtering so it can NEVER re-expose data the viewer cannot see.

Purpose: Game UI authors add visual polish without touching the engine; the engine stays UI-agnostic
(no value-bearing `$`-props); Phase 91's redaction is preserved by construction.
Output: pure `presentation.ts` resolver (with the `__hidden` security guard) + the provide/inject
chain + overlay-first rendering in the visual renderers, fully tested.

SECURITY-CRITICAL: This plan carries the dominant phase threat (information disclosure via overlay).
The resolver MUST strip `image`/`stats` for `__hidden` elements; a unit test proves it.
</objective>

presentation.ts (pure module — analog: renderer-registry.ts; no Vue reactivity, no DOM, local GameElement):
  interface PresentationEntry { image?: string; label?: string; stats?: Record<string,unknown>; render?: Component }
  interface PresentationOverlay { byClass?; byName?; byAttribute? }
  resolvePresentation(element {name?,className,attributes?,__hidden?}, overlay?): PresentationEntry | null
    precedence byName > byClass > byAttribute (first match wins, NO merge)
    if element.__hidden → return { label: entry.label, render: entry.render } (strip image+stats)

Provide/inject chain (reactive):
  GameShell: provide('presentation', toRef(props,'presentation'))
  AutoRenderer: provide('presentation', computed(() => props.presentation))
  renderer: inject<ComputedRef<PresentationOverlay|undefined>>('presentation')
