<objective>
Fix the two foundational interaction defects all later gates depend on, each proven by a mounted
component test FIRST (Prove Before Fix):

1. Bug C (the deepest, confirmed root cause): ActionPanel's `filteredChoices` calls the NON-reactive
   `getCurrentChoices()` (ActionPanel.vue:230), which reads a plain `Map.get()` and never tracks
   `snapshotVersion`. When async choices arrive (`fetchChoicesForPick` does `Map.set` + `snapshotVersion++`),
   the footer never re-renders → shows "No options available" despite legal choices. The reactive
   computed `currentChoices` already exists and is exported. Same gap at ActionPanel.vue:264 for
   `getValidElements()` → use `validElements.value`.

2. D-02 footer fallback: change `allCurrentChoicesAnchored` to `return validElements.value.length > 0`
   so the footer is suppressed ONLY when the board can fully drive the pick (element picks with valid
   elements), and STAYS PRESENT for choice-type picks. This flips exactly 4 existing assertions; update
   them to the new semantics (do not delete the tests).

Purpose: Without these two fixes the Checkers destination step and Go Fish ask have no working
interaction surface. This plan is the prerequisite foundation for plans 03 and 04.
Output: A reactive ActionPanel, the D-02 footer rule, reproducing component tests, and a green suite.
</objective>
