# Phase 94.1 Plan 04: Go Fish Ask Fix + PRESENT-02 Guard Summary

**One-liner:** Fixed D-03 anchor filter to treat only notation refs as board-anchored (id-only refs are highlights, not board click targets), closing Bug B (Go Fish rank choices now appear in the ActionPanel footer), and added a component-level PRESENT-02 regression guard.

## What Was Built

### TRACE (Task 1) — Open Question 2 resolved

Traced the Go Fish ask flow and confirmed the Bug B root cause by reading source:

1. **Single action confirmed** (`flow.ts:65-73`): `actionStep({ name: 'ask-step', actions: ['ask'] })` — exactly one action available during a normal turn. Candidate cause (1) (simultaneous second action blocking `tryAutoStartSingleAction`) ruled out. `useBoardActionBridge.ts:262` fires with `actions.length === 1` → calls `startAction('ask')`.

2. **Auto-fill path works** (`useActionController.ts:508-524`): `fetchAndAutoFill(target)` fetches 1 opponent (2-player) → `tryAutoFillSelection(target)` auto-fills → `fetchAndAutoFill(rank)` fetches rank choices → `snapshotVersion++` → `currentChoices.value` populated. Plans 02+03 correctly wired the reactive path.

3. **D-03 filter root cause** (`action-panel-helpers.ts:31`): `filterAnchoredChoices` checked `(c.refs ?? []).length > 0` — ANY choice with refs is treated as "board-anchored" and removed from `filteredChoices`. Go Fish rank choices carry id-only refs `{ ref: { id: cardId }, role: 'target' }` for visual card highlighting. All three rank choices have refs → `hasAnyAnchored = true` → `choices.filter(refs.length === 0)` → returns `[]`. The ActionPanel template (line 987) has `v-else-if="currentPick.type === 'choice' && filteredChoices.length"` — evaluates false → panel shows nothing for the rank pick. Panel text: the action prompt + collected target chip ("Ask another player for cards✕Player 2✕") but no rank buttons.

4. **RESEARCH hypothesis was secondary**: The RESEARCH (plan 02) correctly fixed non-reactive choices in ActionPanel, but Bug B persisted because the D-03 filter removed the rank choices before the template could render them.

### Fix (Task 2) — `action-panel-helpers.ts`

Replaced the `refs.length > 0` anchor check with a `refs.some(r => r.ref.notation !== undefined)` guard:

```typescript
// Before (buggy): any choice with any refs treated as board-anchored
const hasAnyAnchored = choices.some(c => (c.refs ?? []).length > 0);

// After (fixed): only notation refs are true board anchors
const isNotationAnchored = (c: ChoiceWithRefs): boolean =>
  (c.refs ?? []).some(r => r.ref.notation !== undefined);
const hasAnyAnchored = choices.some(isNotationAnchored);
```

This preserves Checkers destination behavior (notation refs → filtered from panel → board is the selection surface) while keeping Go Fish rank choices in the panel (id-only refs → not anchored → footer shows rank buttons).
